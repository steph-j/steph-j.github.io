Design to Web

Sat 2Apr16

Responsive Site

Having a site that is responsive means that it expands and contracts as the screen size changes. What had 3 columns on a big screen would automatically turn into 1 column on a phone screen so that the text can be easily read.

Mobile First Design

This is important because at the moment phones are not as powerful as desktops. You want the default load to be small so that the mobile device won't be overloaded by information and cause lag. As the screen size increases more can be added in.

A greater number of people are accessing the internet from their phones so we need to be aware of how the information is appearing to them and put the most important information first.

Frameworks

A grid system made up of columns and rows. Skeleton is a framework that is split into 12x12. This is very helpful in placing different items on the site. Sometimes it can be restrictive.

Wireframe

This is a blueprint on a grid of how the page should apear when it is online. We use it to plan the page so that when we start the html doc we know how many columns we need for each section.

Difficulties with Wireframe

I find positioning the most challenging part of wireframes. It is just a matter of trial and error sometimes to find out how to make something move. I believe, just this morning, that I may now understand. I simplified my design as I went along when I saw how long everything was taking. I know that I will have more time in the future to play around with it.
